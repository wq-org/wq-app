Rich Text Editor — Toolbar Reference

Every button in the floating toolbar, what it does, its keyboard shortcut, and how it maps to HTML output. Use this as a guide when building or documenting lesson content in wq.

01 — Overview

The Floating Toolbar

When you select text in the editor, a floating toolbar appears above your selection. It contains formatting controls grouped by function: text style, structure, linking, and annotation.

Toolbar — as it appears on text selection

Click the color chip (last icon) to toggle the color picker.

02 — Controls

Every button, explained

Each control applies a specific HTML mark or action to the selected text. Here is what each one does and when to use it.

B

Bold

Emphasises the selected text using strong importance. Use for key terms, warnings, or critical steps a learner must not miss.

⌘ B <strong>
I

Italic

Marks text with stress emphasis or alternates tone — ideal for titles of works, foreign terms, or subtle clarifications within a sentence.

⌘ I <em>
U

Underline

Draws a line below text. Use sparingly — readers associate underline with links. Best reserved for proper nouns, annotations, or named entities.

⌘ U <u>
S

Strikethrough

Draws a line through text to indicate deletion or a superseded value. Useful in wound-care content to show "old approach → new approach" contrasts.

⌘ ⇧ X <s>

Inline Code

Wraps text in a monospace code style. In an educational game context, use it for clinical codes (ICD-10), product names, or exact identifiers learners must recall.

⌘ E <code>

Hyperlink

Turns selected text into a clickable link. Opens a URL input dialog. Use to reference clinical guidelines, external resources, or internal lesson nodes.

⌘ K <a href>

Comment / Annotation

Attaches an inline pedagogical note to the selected text — visible to instructors and editors, not to learners in play mode. Stored as a Tiptap extension mark.

Instructor-only
·

Text Color

Opens a 15-color palette to apply a custom foreground color to the selection. Renders as an inline style="color:..." or a custom Tiptap color mark depending on your extension config.

05 — Output

HTML output vs. plain text

The editor produces semantic HTML. Here's what a formatted sentence looks like as HTML versus its rendered appearance.

HTML output (stored in Supabase)
<p>Always apply a <strong>moist wound environment</strong> — <em>never</em> let the wound bed dry out. Reference code: <code>L89.313</code>. See <a href="#guideline">EPUAP guideline</a>.</p>
Rendered result (learner view)

Always apply a moist wound environment — never let the wound bed dry out. Reference code: L89.313. See EPUAP guideline.
