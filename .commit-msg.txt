fix(lexical-editor): use default Prism highlighting and simplify code blocks

**Problem:**
- Custom Prism grammar registration crashed the editor with Cannot set properties of undefined (setting 'string').
- WQ-maintained language imports were fragile under Vite load order and duplicated @lexical/code-prism defaults.

**Decision:**
- Remove custom languages.ts grammar registration and rely on Lexical's built-in Prism tokenizer.
- Drop the hover language picker; keep a minimal copy-on-hover menu for code blocks.
- Refresh code-block theme for light/dark and add ordered-list depth styling in the editor.

**Changes:**
- src/features/lexical-editor/plugins/code-highlight-plugin/languages.ts: deleted custom Prism grammar list and registration.
- src/features/lexical-editor/plugins/CodeBlockActionMenuPlugin.tsx: added copy-only hover menu for code blocks.
- src/features/lexical-editor/components/Editor.tsx: wire CodeBlockActionMenuPlugin into the editor shell.
- src/features/lexical-editor/plugins/code-highlight-plugin/codeHighlightTheme.css: restyle tokens and dark-mode palette; remove gutter styling.
- src/components/shared/editors/DocumentCodeHighlightPlugin.tsx: register highlighting via @lexical/code-prism.
- src/features/lexical-editor/plugins/blockOptions.tsx: insert tables with $createTableNodeWithDimensions and focus first cell.
- src/components/shared/editors/editorTheme.ts, src/index.css: add ordered-list depth marker classes.
- src/components/shared/pdf-viewer/PdfLinkInsertPopover.tsx: simplify local handlers.
- package-lock.json: lockfile metadata cleanup for optional platform entries.

**Impact:**
- Code blocks highlight without runtime Prism registration errors.
- Less custom syntax-highlighting surface area to maintain.
- Tables insert at the cursor with immediate cell focus; nested ordered lists render 1./a./i. cycles.

**Behavior_Change:**
- BEFORE: Custom 30+ language Prism imports could crash highlighting; hover menu offered language picker + copy.
- AFTER: Default Lexical/Prism languages only; hover menu offers copy; code blocks use updated light/dark styling.

**Trade_offs:**
- No in-editor language picker for now; new blocks still default to TypeScript via block options.
- Fewer Prism grammars than the old custom list (Go, YAML, etc. not registered unless Lexical adds them).

**Verified:**
- npx vitest run src/features/lexical-editor/plugins/code-highlight-plugin/kit-smoke.test.ts passes.
- Dev server reload: open note editor, insert code block, confirm highlighting and copy menu without console errors.
